□ STEP 2: Admin Send Notification

Action:
  Admin Dashboard → Create notification → Send Now

What happens:
  1. POST /api/admin/notifications/push
  2. Creates row in notifications table
  3. Auto-injects data.notification_id = row.id
  4. Queries matching devices
  5. Sends to FCM immediately

Expected:
  "Sent to X devices" (X >= 1)

Verify in Supabase:
  SELECT id, title, status, sent_count
  FROM notifications
  ORDER BY created_at DESC
  LIMIT 1;

Pass Criteria:
  □ Row created with correct title/body
  □ status = 'sent'
  □ sent_count >= 1
  □ created_at is recent

🚨 COMMON ISSUES & QUICK FIXES

Issue: "Token not found in database"
  → Check Next.js API is deployed
  → Verify /api/push/register endpoint works
  → Check Supabase connection

Issue: "Notification not received"
  → Verify token in database (last_seen_at recent)
  → Check device notifications enabled
  → Verify FCM credentials
  → Check admin query matches your device (country/topic)

Issue: "data.notification_id missing"
  → This should NEVER happen with admin sends
  → Check Next.js auto-injection logic
  → Verify notifications row was created

Issue: "Rate-limiting not working"
  → Verify POST /api/push/send endpoint
  → Check rate-limit storage (database/Redis)
  → Try with fresh user account

Issue: "Routing not working"
  → Check data.screen value
  → Verify routing logic in onMessageOpenedApp
  → Ensure data.entity_id is passed correctly
