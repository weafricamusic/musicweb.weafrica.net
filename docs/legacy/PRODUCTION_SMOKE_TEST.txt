🔹 STEP 1 — Consumer App (Flutter)

After Firebase login:

1. Get Tokens:
   • Firebase ID Token (for auth)
   • FCM Device Token (for notifications)

2. Register Device:

   Endpoint: POST /api/push/register
   Headers:  Authorization: Bearer <firebase_id_token>
   
   Request Body:
   {
     "token": "FCM_DEVICE_TOKEN",
     "platform": "android",
     "device_id": "unique-device-id",
     "country_code": "mw",
     "topics": ["all", "consumers"]
   }

   ✅ This hits your Next.js API route
   ✅ Next.js API inserts/updates row in Supabase
   ✅ Response: 200 OK with token data

Flutter Code:
```dart
final helper = PushSmokeTestHelper();
final tokens = await helper.collectTokens();
final result = await helper.registerDeviceToken(
  idToken: tokens['id_token'],
  fcmToken: tokens['fcm_token'],
  platform: tokens['platform'],
  deviceModel: tokens['device_model'],
  countryCode: 'mw', // or 'gh', 'ng', etc.
);
```

Go to: Supabase Dashboard → SQL Editor

Run this query:
```sql

  token,
  user_uid,
  topics,
  country_code,
  platform,
  last_seen_at

🔹 STEP 4 — Receive on Device

Check device for notification:

Foreground (App Open):
  → FirebaseMessaging.onMessage fires
  → Console shows: "🔔 Foreground message received"
  → You can show custom in-app UI

Background (App Closed) / Tap:
  → Native notification appears
  → Tap → FirebaseMessaging.onMessageOpenedApp fires
  → App opens and routes to screen based on:
     • data.screen (target screen)
     • data.entity_id (track/post/battle ID)
     • data.notification_id (ALWAYS present = row.id from admin)

```

🔧 TROUBLESHOOTING

Problem: "Failed to register device token"
  → Check Next.js API route is deployed
  → Verify endpoint URL is correct
  → Check Firebase ID Token is valid

Problem: "Token not found in Supabase"
  → Verify Next.js API successfully inserted row
  → Check Supabase connection in Next.js
  → Query by token field, not fcm_token

Problem: "Topics not showing as array"
  → Verify Next.js sends topics as JSONB
  → Check column type is JSONB, not TEXT

Problem: "user_uid is null"
  → Verify Next.js extracts UID from Firebase token
  → Check Authorization header is sent
